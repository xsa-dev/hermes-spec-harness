# Fix Specification

## Goal

Describe the bounded fix and the user-visible result.

## Context / Evidence

List the observed failure, reproduction notes, logs, and file references. Redact sensitive values.

## Scope

Define the exact behavior and files expected to change.

## Out of scope

List related work that must not be included in this change.

## Expected files

List likely files or directories touched by the implementation.

## Acceptance criteria

- The described bug or gap is resolved.
- Existing behavior outside scope is preserved.
- Verification commands pass or skipped reasons are documented.

## Verification

List commands, manual checks, and expected results.

## Secret safety

Do not print or commit secrets, tokens, `.env` values, authorization headers, private keys, or private endpoints.

## Archive plan

Archive this OpenSpec change after verification and before the commit boundary.
