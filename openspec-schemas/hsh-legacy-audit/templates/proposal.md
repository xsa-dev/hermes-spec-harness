# Legacy Audit Proposal

## Goal

Create a read-only baseline understanding of the project before implementation work begins.

## Constraints

Do not modify source code, dependency files, generated files, or tests during the audit.

## Audit boundaries

Define included repositories, packages, services, and excluded systems.

## Risk framing

Identify known operational, security, architecture, dependency, and delivery risks.

## Deliverables

- Project inventory.
- Subsystem map.
- Command matrix.
- Risk register.
- Modernization roadmap.
